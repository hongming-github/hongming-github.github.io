Mac下如何用hexo搭建自己的博客

8月 18, 2015 in hexo

一、安装Node.js和Git

非常简单，请自行百度 or Google。

二、进入终端，输入如下命令npm install hexo-cli -g。

但是呢，由于特殊原因，有时候这条命令不好用，所以，万能的淘宝团队干了件好事，通过淘宝镜像，我们可以用如下命令cnpm install hexo-cli -g进行安装。以下所有npm命令同理。

三、创建博客

通过以下四条命令可以完成博客的创建
1.hexo init blog
2.cd blog
3.npm install
（注意：在这个步骤的时候可能出现以下错误
{ [Error: Cannot find module ‘./build/Release/DTraceProviderBindings’] code: ‘MODULE_NOT_FOUND’ }
{ [Error: Cannot find module ‘./build/default/DTraceProviderBindings’] code: ‘MODULE_NOT_FOUND’ }
{ [Error: Cannot find module ‘./build/Debug/DTraceProviderBindings’] code: ‘MODULE_NOT_FOUND’ }）
此时，我们只需要输入npm install hexo –no-optional就可以了
4.hexo server
此时，我们就可以通过在浏览器中输入localhost:4000来访问自己的博客了。

四、选择一个主题

创建好博客后，会有一个默认主题供我们使用。如果想要换成其他的，可以在网上搜索，然后通过主题作者的说明文档来学习如何使用和配置。

五、使用GitHub Pages建立博客

在github上建立仓库，仓库的名字一定是username.github.io。

六、部署到github（把个人博客和github连在一起）

首先，在Hexo 3.0版本后deploy git 被分开的，所以需要安装，输入
npm install hexo-deployer-git –save命令
然后，到配置_config.yml文件中找到
deploy:
type:
然后修改为:
deploy:
type: git
repo: git@github.com:username/username.github.io.git
branch: master

注意：
1.每个冒号后面都有一个空格！
2.repo必须是SSH形式的url，是仓库的路径，可以在github上获取
然后输入以下命令进行部署：
hexo clean
hexo generate（可以简写为hexo g）
hexo deploy（可以简写为hexo d）
成功后，如果能成功访问username.github.io到你的博客，那说明你已经配置完成了！

写在最后：

技术更新太快，要时刻关注官方网站的技术文档，语法，命令随时都有可能更新，个人教程很多都已经过期了。
